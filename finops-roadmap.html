FinOps Calculator Roadmap (2026)

Public summary of quarter-by-quarter feature evolution for the FinOps and Cloud Economics Calculator, expanding toward a converged technology economics decision engine.

Main site Calculator documentation Source repository

Execution backlog is managed in GitHub Issues/Project under duksh/finops-calculator. This page mirrors the public roadmap summary.

Q1 - Foundation and Trust

Core model hardening and explainability upgrades.

  • Unified Input Model v2
  • Assumption Versioning
  • Scenario Compare (Baseline vs Optimized)
  • Formula Trace Panel
  • Input Quality Guardrails
Calculator impact: faster onboarding, fewer invalid runs, and much higher trust in outputs.

Q2 - ITAM and Portfolio Convergence

Expand from cloud-only modeling to total technology economics.

  • ITAM Cost Overlay
  • Portfolio View (multi-workload)
  • Forecast Confidence Bands
  • Recommendation Prioritization v2
Calculator impact: more accurate pricing floor/margin outcomes and stronger planning decisions at portfolio level.

Q3 - Governance Expansion

Add GreenOps and AI economics governance capabilities.

  • GreenOps Carbon Layer
  • AI Economics Module
  • Policy and Threshold Engine
  • Anomaly and Drift Alerts
Calculator impact: supports sustainability goals, AI workload economics, and policy-based cost control.

Q4 - Decision Engine and Enterprise Readiness

Operationalize recommendations and track realized value.

  • Decision Playbooks
  • Value Realization Tracker
  • MCP Workflow Automation
  • Audit and Role-based Views
Calculator impact: transforms the tool into a continuous decision and optimization platform.
